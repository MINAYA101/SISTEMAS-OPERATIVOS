Concurrencia vs Paralelismo

Basado en la explicación del video solicitado

Concurrencia

"Coexistencia de tareas"

Es la capacidad de procesar más de un hilo de ejecución al mismo tiempo. Los procesos conviven y progresan, pero no necesariamente se ejecutan en el mismo instante exacto.

  • Funciona en 1 solo núcleo (monoprocesador).
  • Se gestiona mediante alternancia de tiempo (Time Slicing).
  • Palabra clave: Estructura.

Paralelismo

"Ejecución simultánea real"

Es la ejecución literal de múltiples procesos al mismo instante físico. Requiere hardware específico para soportarlo.

  • Requiere Múltiples núcleos (Multicore).
  • Las tareas ocurren exactamente a la vez.
  • Palabra clave: Ejecución.

¿Cómo se ve en el procesador?

Escenario Monoprocesador (Concurrencia)

CPU Única
Tarea A
Tarea B
Tarea A
Tarea B
Tarea A
Tiempo T1 T2 T3 T4 T5

Aunque hay dos tareas, la CPU alterna entre ellas. Nunca se ejecutan a la vez, pero el usuario percibe que sí.

Escenario Multiprocesador (Paralelismo)

Core 1
Core 2
Tarea A (Core 1) - Ejecutando...
Tarea B (Core 2) - Ejecutando...

Ambos procesadores trabajan al 100% simultáneamente. Tarea A y Tarea B ocurren en el mismo instante físico.

La Relación Lógica

Paralelismo Concurrencia

Si es paralelo, implica que los procesos coexisten, por tanto, también es concurrente.

Concurrencia Paralelismo

La concurrencia NO garantiza paralelismo. Depende del hardware disponible.

Ejercicio Práctico: Código Java

class Tarea extends Thread {
    public void run() {
        // Imprime 1 al 500
    }
}

public static void main(String[] args) {
    Tarea t1 = new Tarea();
    Tarea t2 = new Tarea();
    
    t1.start(); // Lanza hilo 1
    t2.start(); // Lanza hilo 2
}

¿Es Concurrente o Paralelo?

Veredicto 1

¡SÍ es Concurrente!

Se crean dos hilos y se lanzan a la vez. Coexisten.

Veredicto 2

¿Es Paralelo? DEPENDE

No lo sabemos a priori.
Si tienes 1 CPU → NO es paralelo.
Si tienes Multi-CPU → Probablemente SÍ.
